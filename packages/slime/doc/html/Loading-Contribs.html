9.1 Loading Contrib Packages

Contrib packages aren't loaded by default. You have to modify your setup a bit so that Emacs knows where to find them and which of them to load, i.e. you should add the contrib directory to Emacs' load-path and call slime-setup with the list of package-names. For example, a setup to load the slime-scratch and slime-editing-commands looks like: 

     (add-to-list 'load-path ".../slime")          ; path for core
     (add-to-list 'load-path ".../slime/contrib")  ; path for contribs
     (require 'slime-autoloads)
     (slime-setup '(slime-scratch slime-editing-commands))

After starting SLIME, the commands of both packages should be available.
